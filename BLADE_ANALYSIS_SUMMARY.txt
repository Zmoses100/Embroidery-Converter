5. @SECTION DIRECTIVE PATTERN (IMPORTANT)
═══════════════════════════════════════════════════════════════════════════════

The "mismatch" in @section/@endsection is NOT an error - it's the correct pattern:

Pattern in Each Content View:
  Line 1: @extends('layouts.app')
  Line 2: @section('title', 'Page Title')  ← Inline declaration
  Line 3: @section('content')               ← Main content block
  ...
  Last:   @endsection                       ← Closes @section('content')

The inline @section('title') doesn't need closing because it's one-line.
The @endsection closes only the content section.

Example (resources/views/auth/login.blade.php):
  @extends('layouts.guest')
  @section('title', 'Login')           ← No closing needed
  @section('content')                  ← Opens main content
    ... content ...
  @endsection                          ← Closes main content
  
This is the CORRECT Laravel Blade pattern.
